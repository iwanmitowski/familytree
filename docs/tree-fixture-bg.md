# Тестово семейство за дървото

Стъпки за ръчна проверка на визуализацията (Task 31) в dev среда. Използва
admin потока — така се създават канонични хора + потвърдени връзки, които
дървото показва.

## Предпоставки

- Dev услугите вървят: `docker compose -f docker-compose.dev.yml up -d` (Postgres),
  `npm run dev` в `services/api` и `apps/web`.
- Admin достъп (Google акаунт в allowlist-а) към `/admin`.

## Seed през админ панела

1. **Хора** (`/admin/people` → „Нов човек"): създайте 3 поколения. Пример:
   - Дядо (1900–1970, публично), Баба (1905–1980, публично)
   - Баща (1930–2001, семейно), Майка (1935, жива, семейно)
   - Аз (1958, жив, частно), Брат (1960, жив, частно)
   - Син (1985, жив, частно)
2. **Съюзи** (страница на човек → таб „Съюзи" → „Нов съюз" + „Добави партньор"):
   - Дядо ↔ Баба; Баща ↔ Майка; Аз ↔ (партньор по избор).
3. **Връзки** (таб „Връзки" → „Добави родител/дете" → потвърдете):
   - Дядо/Баба → Баща; Баща/Майка → Аз и Брат; Аз → Син.
   - Натиснете „Потвърди", за да станат `confirmed` (дървото показва
     потвърдените; предложените се рисуват с прекъсната линия).

Алтернатива: минете през въпросник → `/admin/submissions` → „Резолюция"
(създай/свържи) → „Връзки" (Потвърди) → „Маркирай като обработена" (Task 27).

## Проверка

- **Admin дърво** (`/admin/tree`): изберете корен (напр. „Аз") → многопоколенно
  дърво с ELK подредба (предци нагоре, потомци надолу), съюзни възли между
  партньорите, без дублиран човек. Бутоните „Побери" и „Центрирай" работят;
  клик върху възел показва информационен панел.
- **Публично дърво** (`/tree?root=<id на Дядо>`): живите членове са маскирани
  („Жив член на семейството", без години); починалите публични/семейни показват
  име и години. Търсенето не намира живи хора.
- Подредбата не блокира таба (ELK върви във Web Worker; при липса на worker —
  на главната нишка).
