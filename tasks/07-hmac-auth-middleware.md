# Task 07: HMAC service authentication middleware

**Depends on:** 05 · **Size:** L · **Spec:** idea.md §4 (entire section), §23 (HMAC tests)

## Goal
Implement the HMAC request authentication layer for all `/v1/internal/*` endpoints, plus a written canonical-signing spec and cross-language test vectors that the BFF (Task 09) must also satisfy.

## Requirements
1. `contracts/hmac.md` — precise spec both implementations follow:
   - Headers (idea.md §4): `X-Service-Id`, `X-Request-Timestamp` (RFC3339 UTC), `X-Request-Nonce` (UUIDv4), `X-Idempotency-Key`, `X-Body-SHA256` (lowercase hex of raw body; empty body = hash of empty string), `X-Actor-Id`, `X-Actor-Role`, `X-Signature`.
   - Canonical payload: newline-joined, exactly: `HTTP_METHOD` (uppercase), `REQUEST_PATH` (escaped path; if a query string exists append `?` + raw query), `TIMESTAMP`, `NONCE`, `IDEMPOTENCY_KEY` (empty string when absent), `BODY_SHA256`, `ACTOR_ID`, `ACTOR_ROLE`.
   - Signature: `hex(HMAC-SHA256(SERVICE_HMAC_SECRET, canonicalPayload))`.
2. `contracts/hmac-test-vectors.json`: ≥6 vectors generated by the Go implementation — GET no body, GET with query string, POST with JSON body, POST with Cyrillic (UTF-8) body, empty idempotency key, actor variations. Each vector: secret, all inputs, canonical payload string, body hash, expected signature.
3. Go middleware in `internal/auth` applied to everything under `/v1/internal` (idea.md §4 checks, in order):
   1. Known `X-Service-Id` (config: `SERVICE_ID`, `SERVICE_HMAC_SECRET`).
   2. Timestamp parseable and within ±5 minutes.
   3. Nonce not seen: insert into `service_request_nonces` (expiry now+10min); unique-violation ⇒ reject.
   4. Signature valid via **constant-time comparison** (`hmac.Equal`).
   5. Idempotency (mutating methods with a key): if key exists with same `request_hash` (hash of canonical payload) ⇒ replay stored response (status + body) without re-executing; same key + different hash ⇒ 409. Provide a response-capture wrapper that stores successful responses.
   6. Actor: place `actorID`/`actorRole` into request context (trusted only because signed). Helper `RequireRole("admin")` middleware for admin-only endpoints.
4. Failures return generic 401 `{"error":{"code":"unauthorized","message":"authentication failed",...}}`. Log details at debug level **without** the secret or full signature (idea.md §4.7–4.8).
5. Background janitor goroutine: hourly delete of expired nonces and idempotency keys (idea.md §4).
6. Exemptions: `GET /health` (public, reveals nothing). Decision (document in `contracts/hmac.md`): `GET /ready` stays unauthenticated but returns only `ok`/`unavailable`.
7. Config additions → `services/api/.env.example`.

## Acceptance criteria
- All §23 auth cases behave correctly: valid request passes; bad signature, expired timestamp, future timestamp, reused nonce all → 401; idempotent replay returns the stored response; same key different body → 409; tampering with `X-Actor-Role` after signing → 401.

## Verification
- Unit tests for canonical payload construction (all vector cases) + the full middleware matrix above; integration tests for nonce uniqueness and idempotency storage.
- A test asserting every vector in `contracts/hmac-test-vectors.json` verifies against the middleware.
- Standard Go verification + `go test -tags=integration ./...`.
- Commit as `task-07: hmac auth middleware`.
